El elemento <img>

El elemento <img> se usa para incrustar imagenes en documentos HTML

Sisntaxis minima correcta

      
        <img src="ruta/a/la/imagen.jpg" alt="Descripción de la imagen">

El atributo <alt> proporciona texto alternativos cuando la imagen no puede mostrarse y es clave para accesibilidad.

Mejores paracticas para <img>

  • Siempre incluye el atributo <alt> con una descripción concisa de la imagen.
  • Usa rutas relativas para las imágenes cuando sea posible. Esto hace que el sitio sea más fácil de mantener.
  • Usa atributos width y height para especificar las dimensiones de la imagen. Esto ayuda a prevenir cambios en el diseño mientras la imagen se carga.

Priority Hints en imagenes

Priority Hints (Fetch Priority) permiten indicar la prioridad relativa de descarga de un recurso, sugiriendo al navegador sobre que recursos carga antes o despues.

loading: cuando cargar

Valores:

Valor Significado
lazy La imagen se carga cuando se necesita
eager La imagen se carga inmediatamente
(default) El navegador decide cuando cargar la imagen 
      
        <img src="imagen.jpg" alt="Descripción" loading="lazy">

El atributo <loading> controla el comportamiento de carga (lazy/eager)

fetchpriority: prioridad de descarga

Valores:

Valor Significado
high Indica que el recurso es prioritario
low Indica que el recurso no es prioritario
(default) El navegador decide cuando cargar la imagen 
      
        <img src="imagen.jpg" alt="Descripción" fetchpriority="high">

El atributo <fetchpriority> indica al navegador cómo priorizar la descarga de un recurso.

<img> vs <figure>

El elemento <img> es el medio básico para mostrar una imagen sin contenido adicional

El elemento <figure> representa contenido autocontenido que puede moverse sin afectar el flujo del documento 

      
        <figure>
          <img src="imagen.jpg" alt="Descripción">
          <figcaption>Descripción de la imagen</figcaption>
        </figure>

<figure> se usa para organizar semánticamente imágenes con contexto, mientras <img> es solo el recurso visual.

🎧 <audio> - Incrustar audio

Ejemplo basico recomendado

      
        <audio controls>
          <source src="musica.mp3" type="audio/mpeg">
          <source src="musica.ogg" type="audio/ogg">
          Tu navegador no soporta audio HTML.
        </audio>

✅ controls muestra controles (play, pause, volumen, etc

✅ <source> permite multiples formatos

✅ Texto fallback para navegadores antiguos

el Navegador selecciona automaticamente el primer formato que puede reproducir.

Atributos comunes de <audio>

Atributo Descripción
controls Muestra controles de reproducción
autoplay Reproduce automáticamente el audio
loop Reproduce el audio en bucle
muted Silencia el audio por defecto
preload Indica cómo debe precargarse el audio

📹 <video> - Incrustar video

Ejemplo basico recomendado

      

        <video controls width="640" height="360" poster="preview.jpg">
          <source src="video.mp4" type="video/mp4">
          <source src="video.webm" type="video/webm">
          Tu navegador no soporta video HTML.
        </video>

✅ controls obligatorio para UX

✅ width / height evitan layout shift

✅ poster muestra imagen previa

El atributo poster define una imagen que se muestra antes de reproducir el video.

Atributos comunes de <video>

Atributo Descripción
controls Muestra controles de reproducción
autoplay Reproduce automáticamente el video
loop Reproduce el video en bucle
muted Silencia el video por defecto
poster Indica la imagen que se muestra antes de reproducir el video
width Ancho del video
height Altura del video
playsinline Evita fullscreen en móvil

¿Qué es CSP (content Security Policy)?

CSP permite a los desarrolladores especificar qué fuentes de contenido son confiables, ayudando a mitigar ataques como XSS (Cross-Site Scripting) 7 clickjacking.

¿Por qué CSP es clave para iframe?

Porque los <iframe> pueden:

  • Ejecutar scripts de terceros
  • Mostrar contenido externo
  • Intentar redireccionar la página padre
  • Ser usado para clickjacking

CSP te permite decidir explícitamente:

  • Qué iframes se pueden cargar
  • Desde dónde
  • Quién puede incrustar tu sitio (Frame Ancestors)

<iframe>: qué es y por qué es sensible

<ifram> permite incrustar otro documento HTML dentro del documento actual

Ejemplo:

      
        <iframe src="pagina.html" width="640" height="360"></iframe>

⚠️ PELIGRO: Si no usas CSP, cualquier sitio puede poner tu página en un iframe (clickjacking). Por eso CSP es obligatorio para iframes.

CSP para iframe: frame-src

La directiva frame-src especifica las fuentes válidad para cargar iframes.

Ejemplo:

      
        <!DOCTYPE html>
        <html>
        <head>
          <meta charset="UTF-8">
          <title>Documento seguro</title>
          <meta http-equiv="Content-Security-Policy" content="frame-src https://www.youtube.com;">
        </head>
        <body>
          <iframe width="560" height="315" src="https://www.youtube.com/embed/dQw4w9WgXcQ" allowfullscreen></iframe>
          <iframe width="560" height="315" src="https://wikipedia.org" allowfullscreen></iframe>
        </body>
        </html>

La directiva CSP es:

      
        Content-Security-Policy: frame-src 'self' https://www.youtube.com;

✔️ Permite iframes solo de tu sitio y youtube.com

❌ bloquea cualquier otro origen

✔️ Si quieres bloquear todos los origenes coloca "none" en lugar de la URL 

      
        <meta http-equiv="Content-Security-Policy" content="frame-src 'none';">

sandbox en <iframe> (HTML)

El atributo sandbox impone restricciones de seguridad adicionales al contenido del iframe.

Máxima restricción:

      
        <iframe sandbox src="pagina.html"></iframe>

Esto automáticamente:

  • Bloquea scripts
  • Bloquea formularios
  • Aísla el origen
  • Bloquea popups
  • Bloquea navegación superior

Permisos granulares

      
        <iframe 
          sandbox="allow-scripts allow-same-origin" 
          src="pagina.html">
        </iframe>

Permite:

  • scripts
  • formularios
  • no permite acceso al DOM padre
  • No permite navegar a la página principal

allow-same-origin trata el iframe como si fuera del mismo origen, lo cual reduce el aislamiento, solo se recomienda para contenido interno y confiable.

CSP sandbox vs <iframe sandbox>

CSP <iframe sandbox>
Define Política de seguridad del documento Restricciones del iframe
Alcance Todo el documento Solo el contenido del iframe
Uso Recomendado en Head Atributo HTML en el iframe

En la práctica:

  • CSP define quién puede cargar y quién puede incrustar
  • iframe sandbox define qué puede hacer el contendio cargado

Buenas prácticas reales (production-grande)

✅ Siempre define frame-src en tu CSP

✅ Usar frame-ancestors para prevenir clickjacking

✅ Aplicar sandbox a iframes de terceros

✅ Usa el pricipio de mínimo privilegio

✅ Nunca confíes ciegamente en embeds externos
